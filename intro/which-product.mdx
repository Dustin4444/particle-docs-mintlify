**Particle Network** has five core services. All of them are free to use and independent while compatible with each other. The below guides will help you select the best integration for your project:

---

## Particle SDKs

| Product | What it does | Section |
|---------|-------------|---------|
| **Universal Accounts** | One account + unified balance across all chains. Users never bridge. Supports 7702 compatible wallets. | [Universal Accounts →](/universal-accounts/overview) |
| **Particle Auth** | Social-login wallet creation (Google, Twitter, Discord, etc.) | [Particle Auth →](/social-logins/auth/introduction) |
| **Particle Connect** | Wallet aggregator — one modal for social logins and web3 wallets. | [Particle Connect →](/social-logins/connect/introduction) |
| **AA SDK** | ERC-4337 bundler + paymaster infrastructure for full control over smart accounts | [AA SDK →](/aa/introduction) |
| **Universal Deposit SDK** | Accept deposits from any chain — funds auto-bridge to your destination as USDC | [Universal Deposit →](/universal-accounts/ua-reference/universal-deposit/overview) |

---

## Decision table

| | Universal Accounts | Particle Auth | Particle Connect | AA SDK | Universal Deposit |
|---|---|---|---|---|---|
| **What users get** | One account + unified balance across chains | Social-login wallet, no seed phrase | One modal for any wallet | Smart account with gas sponsorship + batching | Deposit from any chain, lands as USDC on your chain |
| **What you integrate** | Universal Accounts SDK | Particle Auth SDK | Particle Connect SDK | AA SDK + bundler/paymaster config | Universal Deposit SDK |
| **Best combined with** | Particle Auth or Connect as the signer | Universal Accounts or AA SDK | Universal Accounts or AA SDK | Any signer (Auth, Connect, or external EOA) | Any wallet or auth provider |

<Tip>
  **Can I use Universal Accounts without Particle Auth?**

  Yes. The Universal Accounts SDK is provider-agnostic. You can use Particle Connect, Web3Modal, RainbowKit, or any signer that provides an EOA. Particle Auth is optional.
</Tip>
